# RollingPacer – 10일 자동 소멸 롤링페이퍼 웹서비스 기획 문서

## 1. 서비스 개요

RollingPacer는 **로그인 없이 즉시 생성하고 공유할 수 있는 익명 롤링페이퍼 웹서비스**이다.  
사용자는 하나의 링크를 만들어 여러 사람이 메시지를 남길 수 있으며,  
모든 롤링페이퍼는 **생성 시점 기준 10일 후 자동으로 완전히 삭제**된다.

본 서비스는 장기 운영이나 사용자 계정 축적을 목표로 하지 않으며,  
**최소 기능 · 최소 관리 · 자동 소멸**을 핵심으로 설계한다.

---

## 2. 서비스 목표

- 누구나 10초 안에 롤링페이퍼 생성
- 로그인, 인증, 개인정보 수집 없음
- 텍스트 기반의 가벼운 메시지 작성
- 광고 기반 단기/중기 수익
- 운영자 개입 없이 데이터 자동 정리
- 서비스 중단 시 별도 조치 없이 자연 종료 가능

---

## 3. 서비스 운영 원칙 (고정)

1. 회원가입 / 로그인 기능 없음
2. 이메일, 이름, 닉네임 등 개인정보 입력 없음
3. 텍스트 메시지만 허용 (이미지, 파일 업로드 불가)
4. 모든 롤링페이퍼는 **10일 후 자동 삭제**
5. 생성 이후 수정 / 삭제 기능 제공하지 않음
6. 관리자 페이지 없음
7. 서버 유지보수 개입 없음

---

## 4. 사용자 플로우

### 4.1 롤링페이퍼 생성

1. 사용자가 메인 페이지 접속
2. (선택) 제목 입력
3. [롤링페이퍼 만들기] 버튼 클릭
4. 즉시 고유 URL 생성 및 이동

입력 항목:
- 롤링페이퍼 제목 (최대 40자, **선택 사항**)
  - 예: "OO이의 생일", "졸업 축하해", "환영합니다" 등
  - 제목 미입력 시 기본값 없이 빈 상태로 표시

---

### 4.2 롤링페이퍼 공유

- 생성된 URL을 복사하여 자유롭게 공유
- 별도의 초대 / 권한 개념 없음

URL 형식:
`/r/{8자리 영문+숫자 랜덤 문자열}`

---

### 4.3 메시지 작성

- 롤링페이퍼 URL에 접속한 누구나 작성 가능
- 입력 항목:
  - 메시지 본문 (최대 500자)
- 익명 고정
- 이모지 입력 허용
- 줄바꿈 허용

---

### 4.4 메시지 조회

- 메시지는 작성 순서대로 위에서 아래로 표시
- 페이지 새로고침 시 전체 메시지 다시 렌더링
- 실시간 통신(WebSocket) 사용하지 않음

---

## 5. 자동 삭제 정책 (확정)

- 롤링페이퍼 생성 시점 기준 **10일 후 자동 삭제**
- 해당 롤링페이퍼에 포함된 모든 메시지도 함께 삭제
- 삭제된 데이터는 복구 불가
- 삭제 시 사용자에게 별도 알림 제공하지 않음

---

## 5.1 링크 복사 유도 UX 정책

로그인이 없는 서비스 특성상, 사용자가 롤링페이퍼 링크를 잃어버리면 다시 접근할 수 없다.
이를 방지하기 위해 다음과 같은 UX를 적용한다.

### 5.1.1 생성 직후 링크 복사 모달
- 롤링페이퍼 생성 후 페이지 진입 시 **강제 모달** 표시
- 모달 내용:
  - "🎉 롤링페이퍼가 만들어졌어요!"
  - "⚠️ 이 링크를 꼭 저장하세요! 링크를 잃어버리면 다시 찾을 수 없어요"
  - 링크 URL 표시
  - [📋 링크 복사하기] 메인 버튼
  - "나중에 복사할게요" 서브 텍스트 버튼 (클릭 시 모달 닫힘 → 상단 배너 표시)

### 5.1.2 상단 고정 알림 배너
- 링크 복사를 하지 않고 모달을 닫은 경우
- 화면 상단에 **고정 배너** 표시
- "⚠️ 링크를 저장하세요! 나중에 찾을 수 없어요 [복사하기]"
- 복사 완료 시 배너 자동 숨김

## 6. 데이터베이스 설계 (MongoDB 기준)

### 6.1 RollingPaper 컬렉션

```js
{
  _id: ObjectId,
  slug: String,        // 8자리 랜덤 문자열 (unique index)
  title: String,
  theme: String,       // 테마 식별자 (e.g., "theme_1", "christmas_red")
  createdAt: Date,
  expiresAt: Date      // createdAt + 10일
}
```
expiresAt 필드에 TTL 인덱스 적용

TTL 옵션:
```js
expireAfterSeconds: 0
```

### 6.2 Message 컬렉션
```js
{
  _id: ObjectId,
  paperId: ObjectId,   // RollingPaper _id 참조
  content: String,
  createdAt: Date
}
```
RollingPaper 삭제 시 연쇄 삭제 (애플리케이션 레벨 혹은 DB 레벨 처리 필요하지만, 원칙적으로 RollingPaper가 사라지면 조회 불가하므로 자연 소멸 효과)

Message 단독 TTL 없음

## 7. 서버 아키텍처
### 7.1 구조
- Frontend: 정적 페이지 (React 또는 순수 HTML)
- Backend: 서버리스 API
- Database: MongoDB Atlas (TTL 사용)

### 7.2 특징
- Cron, 배치 작업 없음
- 서버가 멈춰 있어도 TTL에 의해 자동 정리
- 운영자가 할 일 없음

---

## 8. 광고 정책
- Google AdSense 사용
- 광고 위치:
    - 메인 페이지 상단 1개
    - 롤링페이퍼 페이지 하단 1개
- 팝업 광고, 인터스티셜 광고 사용하지 않음

---

## 9. 악용 방지 정책
- 메시지 최대 길이 제한 (500자)
- 동일 IP 연속 요청 시 간단한 rate limit 적용
- 욕설 필터, 신고 기능 제공하지 않음
- 운영 개입 전제 기능 전부 제거

---

## 10. 법적/고지 문구 (필수)
메인 페이지 및 하단에 다음 문구 고정 노출:

> 본 서비스는 로그인 없이 이용 가능하며,
> 작성된 모든 메시지는 생성 시점 기준 10일 후 자동 삭제됩니다.
> 개인정보는 수집·저장하지 않습니다.

---

## 11. 서비스 종료 전략
- 데이터는 TTL에 의해 자연 소멸
- 도메인 만료 또는 배포 중단
- 별도의 종료 공지 없이 서비스 종료 가능

---

## 13. 디자인 가이드 (강조사항)

### 13.1 디자인 컨셉
- **키워드**: 트렌디, 귀여움(Cute), 깜찍함, 개성 넘치는(Unique), 프리미엄
- **애니메이션**:
    - 정적인 페이지 지양
    - 마이크로 인터랙션 및 귀여운 등장 애니메이션 적극 활용 (Framer Motion 등 권장)
    - 사용자의 행동에 반응하는 생동감 있는 UI

### 13.2 랜덤 테마 시스템
- **동작 방식**: 롤링페이퍼 생성 시, 약 10가지의 준비된 디자인 테마 중 하나가 **랜덤으로 결정**되어 저장됨
- **테마 구성 요소**:
    - 배경색/패턴/그라디언트
    - 폰트 스타일 및 색상
    - 버튼 및 입력창 스타일
    - 장식 요소 (스티커, 캐릭터 등)
- 각 테마는 서로 확실히 구분되는 개성을 가져야 함
