# RollingPacer – 10일 자동 소멸 롤링페이퍼 웹서비스 기획 문서

## 1. 서비스 개요
RollingPacer는 **로그인 없이 즉시 생성하고 공유할 수 있는 익명 롤링페이퍼 웹서비스**이다.
모든 데이터는 **10일 후 자동 삭제**된다.

## 2. 서비스 목표
- 누구나 10초 안에 롤링페이퍼 생성
- 로그인, 인증 없음
- 텍스트 기반 메시지

## 3. 서비스 운영 원칙
1. 회원가입/로그인 없음
2. 개인정보 없음
3. 텍스트만 허용
4. 10일 후 자동 삭제
5. 수정/삭제 불가

## 4. 사용자 플로우 (Frontend Focus)
### 4.1 롤링페이퍼 생성 (메인 페이지)
- 제목 입력 (최대 40자, **선택 사항**)
  - 예: \"생일 축하\", \"졸업 기념\" 등 용도에 맞게 입력
- [생성] 버튼 -> API 호출 -> 결과 URL로 이동

### 4.2 롤링페이퍼 페이지 (/r/{slug})
- 상단: 제목 표시
- 하단: 메시지 목록 (작성 순)
- 메시지 작성 폼 (최대 500자)
- 공유 기능 (URL 복사)

## 7. 서버 아키텍처 (참고)
- Frontend: 정적 페이지 (React/Next.js/HTML 등)
- API: 서버리스 (Endpoint 연동)

## AI 작업 지침 (Frontend)

### 목적
- RollingPacer의 프론트엔드 UI/UX 구현
- 사용자 중심의 직관적이고 빠른 인터페이스 제공

### 작업 단계
1. **프로젝트 세팅**: Web App (React/Next.js 등) 초기화
    - 모던하고 깔끔한 디자인 시스템 적용 (CSS/Tailwind)
    - **라이브러리 추가**: `framer-motion` (애니메이션용), `styled-components` 또는 `tailwind-merge` 등 스타일링 도구
    - **테마 시스템 구축**: 10가지 이상의 랜덤 테마를 적용할 수 있는 구조 설계
2. **페이지 구현**:
    - `Home`: 타이틀, 생성 input, 생성 버튼
    - `PaperView`: 상단 제목, 공유 버튼, 메시지 리스트, 하단 메시지 작성 폼
3. **기능 구현**:
    - API 연동 (Mocking 우선 또는 Backend와 협의)
    - `POST /api/papers` (생성)
    - `GET /api/papers/:slug` (조회)
    - `POST /api/messages` (메시지 작성)
4. **UX/UI 폴리싱**:
    - 반응형 디자인
    - 로딩 상태 처리
    - 에러 처리 (존재하지 않는 페이지 등)

### 주의사항
- **로그인/회원가입 기능 만들지 말 것**
- 이미지는 사용하지 않음 (이모지는 텍스트로 취급)
- **디자인 강조**:
    - **트렌디하고 귀엽고 깜찍하게**: 딱딱한 게시판 느낌 절대 금지.
    - **애니메이션 필수**: 메시지 등장, 버튼 클릭, 페이지 진입 시 귀여운 효과 적용.
    - **랜덤 테마**: 롤링페이퍼마다 다른 분위기가 연출되도록 10개 이상의 테마셋 구현.
- 10일 후 삭제 정책 명시 (UI에 "이 롤링페이퍼는 D-day에 사라집니다" 표기 권장)
